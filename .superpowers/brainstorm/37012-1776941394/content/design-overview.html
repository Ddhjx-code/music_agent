多角色协作流水线 — 架构设计

验证整体架构，确认后进入下一节细节

数据流

输入
MIDI piece + 用户指令
↓
Planner
解析指令 → 生成执行计划
输出: JSON {phases, params}
↓ 计划
Analyst
提取旋律/和声/音域诊断
输出: 结构化诊断报告
↓ 报告
Arranger
编排乐器布局、声部映射
输出: 修改后的 piece + 编排报告
↓ piece + 报告
Harmonist
生成伴奏/和弦走向（可选）
输出: 追加的伴奏 track
↓ piece
Expression
力度调整、时序变化、踏板
输出: 最终 piece
↓ piece
Critic
质量评审 → 通过/打回
最多打回 2 次到对应角色
↓ 通过
输出
最终 MIDI 文件

核心设计原则

  • 每个角色 = 1 个 LLM 调用 + 专用 prompt + 专用工具子集
  • 角色间通过结构化 JSON 报告通信，不是原始文本
  • Critic 有独立的 LLM 调用，评审规则写入 prompt
  • 打回机制：Critic → 具体角色（不是全部重跑）
  • Planner 不参与音乐创作，只做路由和参数传递
