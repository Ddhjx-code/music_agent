角色详细设计

6 个角色的职责、工具、输出格式

1. Planner — 计划控制器

职责：解析用户指令，决定走哪些阶段、传什么参数

LLM 调用：1 次（不循环）

输入：用户指令 + Analyst 预分析报告

输出：JSON {phases: ["analysis", "arrangement", "expression"], params: {...}}

工具：无 — 纯决策，不调用音乐工具

prompt：prompts/planner.md

为什么不用循环

  • 只做路由，一次决策足够
  • 避免 Planner 陷入无意义循环

风险

  • 如果用户指令模糊，可能路由错误
  • 需要 Analyst 预分析提供上下文

2. Analyst — 音乐分析师

职责：提取旋律、检测和声、识别声部角色、音域诊断

LLM 调用：1 次（读取分析报告后做诊断总结）

输入：musicpy piece

输出：JSON {melody: {...}, harmony: [...], voice_roles: {...}, range_issues: [...]}

工具：extract_melody, analyze_harmony, range_check, voice_detection

prompt：prompts/analyst.md

只读不写

  • 不修改 piece，纯分析
  • 产出结构化报告供下游角色使用

注意

  • ABC notation 可作为附加上下文
  • range_check 结果供 Critic 评审

3. Arranger — 编排师

职责：根据 Planner 计划和 Analyst 报告，编排乐器布局、声部映射

LLM 调用：可循环 2-3 次（编排 → 反馈 → 调整）

输入：piece + Analyst 报告 + Planner 参数

输出：修改后的 piece + JSON {arrangement: "...", tracks_added: N, voice_mapping: {...}}

工具：arrange_for_piano, arrange_for_strings, arrange_for_winds

prompt：prompts/arranger.md

有限循环

  • 2-3 次循环允许微调
  • 每次循环后反馈结构化报告

风险

  • 编排工具内部耦合（如 ArrangePiano 内部调用了 Analysis）
  • 需要解耦或重构

4. Harmonist — 和声师

职责：生成伴奏、调整和弦走向

LLM 调用：可循环 2 次

输入：piece + Analyst 和声报告 + Planner 参数

输出：追加的伴奏 track + JSON {style: "...", pattern: "...", notes_added: N}

工具：generate_accompaniment

prompt：prompts/harmonist.md

注意：可选角色，Planner 计划中不包含则跳过

5. Expression — 表情工程师

职责：力度调整、时序变化、踏板事件

LLM 调用：可循环 2-3 次

输入：piece + 前序角色报告

输出：最终 piece + JSON {velocity_changes: N, timing_type: "...", pedal_events: N}

工具：adjust_velocity, apply_timing_variation, add_sustain_pedal

prompt：prompts/expression.md

注意：只在编排完成后执行，不应改变音符结构

6. Critic — 质量评审

职责：最终质量把关，检查音域、和声合理性、力度平衡、整体连贯性

LLM 调用：1 次

输入：最终 piece + 所有角色报告

输出：JSON {passed: bool, issues: [{role: "...", severity: "high|medium|low", description: "...", fix_instruction: "..."}]}

工具：range_check（只读验证）

prompt：prompts/critic.md

打回规则：有 high severity issue → 打回对应角色重做（最多 2 次）
